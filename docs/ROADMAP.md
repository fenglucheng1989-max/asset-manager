# 产品开发路线图

## 阶段 0：MVP 基础闭环（已完成）

目标：交付一个可独立运行、可完成核心资产维护流程的最小可用版本。

- 用户可注册、登录、退出，并在登录过期后回到登录入口。
- 用户可新增、编辑、删除资产账户，维护账户类型、余额、资产/负债归属、净资产统计开关、颜色和备注。
- 首页展示净资产、总资产、总负债、资产趋势入口和账户金额排行。
- 首页进入和返回时自动刷新数据，并展示必要的加载和错误状态。
- 后端支持账户排序能力，前端首页优先按账户金额排行展示。
- 添加账户入口位于账户列表底部，页面标题区保持简洁。
- 账户表单提供金额格式校验、名称长度提示、备注长度提示和离开确认。
- H5 桌面预览以手机宽度居中展示，中文提示和日志可正常显示。
- H5、Android App 使用统一配置管理 API 地址。
- 后端提供认证、账户 CRUD、资产总览、账户排序和后台管理基础接口。
- 本地开发默认使用 H2 文件存储，保留 PostgreSQL profile 作为可选部署形态。
- 后端覆盖认证、账户 CRUD、资产总览和排序核心集成测试。

## 阶段 1：资产管理增强（已完成）

目标：从“当前余额账本”升级到“资产变化可追踪”。

- 支持手动生成资产快照。
- 支持定时生成每日资产快照。
- 基于 `asset_snapshot` 展示净资产趋势。
- 基于快照展示日、月、年维度的净资产折线走势。
- 支持账户金额排行，默认按折合金额从高到低排列。
- 资产趋势页用账户金额排行和占比表达当前资产结构。
- 支持账户归档，避免删除历史账户。
- 支持多币种录入、账户级折算汇率和 CNY 本位币统计。
- 支持用户默认本位币设置。
- 支持账户详情和账户余额变化记录。

## 阶段 2：记账流水（已完成）

目标：让账户余额由日常流水驱动，形成“记一笔、余额联动、账户详情可追溯”的基础记账闭环。

- 支持收入、支出、转账三类核心流水。
- 支持流水分类、标签、备注、发生时间和账户关联。
- 新增流水后自动调整对应账户余额。
- 删除流水后自动撤销该流水对账户余额的影响。
- 转账流水支持转出账户扣减、转入账户增加，并保持总资产不变。
- 账户详情页展示流水带来的余额变化记录。
- 用户端新增“记账”底部导航，提供按周、月、年切换的流水汇总、收入支出统计、筛选和记一笔入口。
- 点击流水可进入编辑，长按流水可删除。
- 后端提供流水分类、流水列表、新增、编辑、删除接口。
- 后端集成测试覆盖支出、删除回滚、账户转账和余额历史。

阶段 2 增强能力：

- 支持按本周、本月、今年及前后周期切换，并支持类型、分类、账户和日期范围筛选流水。
- 支持用户自定义收入/支出分类，系统默认分类不可编辑。
- 支持月度总预算、分类预算和预算预警。
- 支持月度收入、支出、结余、支出分类占比和日趋势报表。
- 后台管理端支持查看全平台流水、筛选流水和月度收支统计。

暂不纳入近期规划：

- 还款、收益等复合场景的专用交互。
- 附件、票据图片和导入能力。

## 阶段 2.5：我的页账户中心整理（进行中）

目标：把“我的”页从登录入口升级为清晰的账户中心，承接资产入口、偏好设置、数据管理、安全账户和关于反馈。

已完成：

- 展示用户头像首字母、用户名、邮箱和注册日期。
- 展示简版净资产卡片，点击返回资产首页，不重复首页完整资产总览。
- 提供资产趋势、账户管理、快照管理入口。
- 支持默认货币设置。
- 预留主题设置、导出数据、备份与同步、清空所有数据、修改密码、生物识别解锁、注销账号、用户协议、隐私政策和意见反馈入口。
- 功能列表去掉左侧单字文字 icon，采用更轻量的设置页列表样式。

近期排期：

- P0：把“快照管理”从跳转资产趋势页拆成独立快照列表页，支持分页查看、手动生成今日快照和重复生成时更新当日快照。
- P0：补齐“账户管理”独立入口语义。当前可回到资产首页管理账户，后续可按体验需要拆出独立账户管理页。
- P1：实现“导出数据”，优先支持账户、流水、预算、快照导出为 CSV；Excel 导出作为增强。
- P1：实现“修改密码”和“注销账号”，注销账号需要二次确认并说明数据处理结果。
- P1：实现“清空所有数据”，后端提供按当前用户清理账户、流水、预算、快照的事务接口，前端必须二次确认。
- P2：实现“主题设置”，优先支持跟随系统、浅色、深色三种模式。
- P2：实现“备份与同步”，先明确本地导出/导入和云端同步边界，再进入开发。
- P2：实现“生物识别解锁”，按 App 端能力接入，H5 仅保留不可用状态。
- P2：补齐用户协议、隐私政策和意见反馈页面。

## 阶段 3：多端体验与管理能力

目标：提升正式使用体验和管理能力。

- Android 自定义图标、启动页和正式包配置。
- iOS 打包适配。
- PWA 安装体验。
- 暗色模式和安全区适配。
- 后台管理支持分页、批量操作、操作日志。
- 后台权限细分为只读运营、财务管理员、超级管理员。

## 阶段 4：工程质量和协作效率

目标：降低维护成本。

- CI 覆盖后端测试、前端构建、H5 产物检查。
- 生成 OpenAPI 文档。
- 支持请求 ID 和关键业务日志。
- 支持前端关键页面截图测试。
- 整理环境变量模板和部署说明。
