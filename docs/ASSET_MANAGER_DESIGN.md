# 资产管家设计文档索引

本文档只保留项目定位、阅读入口和文档拆分规则，避免后续协作时反复加载过大的综合设计文档。

## 项目定位

资产管家是个人资产与负债账户管理工具。当前版本优先解决资产账户维护、净资产总览、账户变化追踪、多币种折算、记账流水基础闭环、多端预览和后台管理。

## 当前阶段

当前处于记账流水阶段完成阶段：

- 后端 Spring Boot API 已可运行。
- 默认使用 H2 文件存储，保留 PostgreSQL profile 用于切换。
- 用户端 H5 已可登录、注册、维护账户、设置默认本位币、查看资产总览、账户详情、余额变化记录、资产趋势、记账流水、流水筛选、分类维护和预算预警。
- 后台管理端已可查看平台概览、用户列表、账户列表和流水管理。
- 本地 `local` profile 会初始化管理员账号：`admin / admin123456`。

## 文档维护规则

- 当前功能变化优先更新 `FEATURES.md`。
- 新需求、阶段规划和优先级变化优先更新 `ROADMAP.md`。
- 技术架构、目录结构、运行方式变化优先更新 `ARCHITECTURE.md`。
- 数据库表、字段、迁移策略变化优先更新 `DATABASE.md`，真实 SQL 以 `backend/src/main/resources/db/migration/` 为准。
- 不再把完整 SQL 大段复制进综合设计文档，避免文档变大并造成 token 浪费。

## H5 体验原则

- 页面内部不重复导航栏已经表达的标题。
- 首页避免常驻解释性文案和非必要按钮，优先展示资产数据和必要操作。
- 列表类操作尽量贴近列表本身；添加入口可放在列表底部，排序采用长按拖动等低干扰交互。
- 手势交互不能影响页面滚动，只有明确进入排序等编辑状态后才接管拖动。
- 首页净资产卡承载资产趋势入口，避免额外入口卡片占用首屏空间。
- “我的账户”是管理列表，不是排行榜；顺序由用户维护，金额占比只作为辅助信息。
- 账本页筛选保持轻量：时间周期收纳到日期标题，类型使用文字 tab，分类作为独立筛选入口；类型为转账时分类置灰禁用。
- 预算信息应先表达状态和风险，再作为入口；预算摘要展示已用、总额、剩余/超支、进度和预警状态。
- 收支结构使用同一卡片内的支出/收入切换，避免多个大卡片连续堆叠造成页面臃肿。
- “我的”页作为账户中心，不堆叠冗长说明；已实现能力直接进入，未实现能力明确提示暂未开放。
- 个人中心的资产摘要只做轻量回访入口，不重复首页完整资产总览。
